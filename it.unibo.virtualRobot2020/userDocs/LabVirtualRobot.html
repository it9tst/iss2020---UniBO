INTRODUCTION

Our company (Unibo) has developed a virtual environment (named WEnv) that includes a virtual robot (named WENV-ROBOT).
The virtual environment has been mainly built by Pierfrancesco Soffritti using the three.js JavaScript library (see ConfigurableThreejsApp ) .

Starting

The reference project for this introduction is it.unibo.virtualRobot2020.
  1. Install Node.js
  2. Open a terminal on it.unibo.virtualRobot2020\node\WEnv\server and execute 
    npm install
  3. In the directory it.unibo.virtualRobot2020\node\WEnv\WebGLScene execute 
    npm install
  4. In the directory it.unibo.virtualRobot2020\node\WEnv\server\src execute 
    node main 8999
  5. Open a browser on 
    localhost:8090

INTERACTION

The WENV-ROBOT accepts commands sent on a TCP connection on port 8999. The format of commands (JSON) is described in the file README.md. For example: 
moveForward : `{ "type": "moveForward", "arg": 300 }`
moveBackward : `{ "type": "moveBackward", "arg": 300 }`
 
turnRight : `{ "type": "turnRight", "arg": 300 }`
turnLeft : `{ "type": "turnLeft", "arg": 300 }`

stop : `{ "type": "alarm" }`
Moreover, the virtual environment sends to the connected client (via TCP on port 8999):
  • data emitted by a sonar included in the scene (scene-sonar) when it detects the robot
  • data emitted by the sonars on board of the robot (robot-sonar), when they detect an obstacle (fixed or mobile) 
webpage-ready : `{ "type": "webpage-ready, "arg": {} }`
sonar-activated : `{
	"type:" "sonar-activated",
	"arg": { "sonarName": "sonarName", "distance": 1, "axis": "x" }
}`
collision : `{
	"type": "collision",
	"arg": { "objectName": "obstacle-1" }
}`

USING THE WENV-ROBOT WITH KOTLIN

Let us report an example of a client written as a Kotlin object that provides a function to send commands to the robot using a TCP socket:

clientWenvTcpObj.kt

This file is also included in project it.unibo.kotlinintro, directory virtualRobotUsage.

The coroutine launched by clientWenvTcpObj.kt waits for the sonar-data sent by the virtual robot environment along the TCP connection; it simply extracts values from the received JSON string and prints these values.
This low-level approach to the usage of a (virtual)robot should be replaced by a higher-level, more technology-independent code, written after an application-oriented (top-down) design.
For such a change, please read it.unibo.kotlinIntro\userDocs\FirstActorRobot.html.
